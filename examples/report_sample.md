# 求职分析（示例）

> 本报告根据 `resume_sample.md` 与 `jd_sample.md` 编写，仅用于展示输出形式。

## 岗位要求

- Python、API 调用、JSON 和文件读写
- 大语言模型、提示词与 Tool Calling 基础
- Git、Markdown 和清晰的项目表达

## 简历匹配证据

- 候选人的两个项目都使用 Python，并涉及模型 API 或文件处理。
- 简历明确提到 Tool Calling、Git 和 Markdown，覆盖 JD 的基础要求。
- 课程资料问答助手包含接口超时和异常返回处理，与岗位的测试改进工作相关。

## 简历改写建议

- 补充本人负责的具体模块，区分个人工作与课程整体成果。
- 说明如何验证输出质量，例如测试样例数量、失败类型和修复方法。
- 如果没有 RAG 实践，不要为了匹配加分项而编造经历。

## 面试准备提示词

- 请解释一次完整的 Tool Calling 循环，以及工具执行失败时如何处理。
- 为什么要限制 Agent 可读写的文件路径？
- 你会怎样评测求职分析报告是否准确且没有编造？

## 下一步行动

1. 为两个项目各准备一段两分钟说明。
2. 整理三个真实失败案例及对应修复过程。
3. 检查简历中的每项技能是否都有项目证据支持。
